🚀 Quick Start Guide

Get up and running in just a few steps:

  1. Build the simulator: 
    cargo build --release
  2. Create a simple program (save as my_circuit.json): 
    [
  {"op": "init", "n": 2},
  {"op": "h", "q": 0},
  {"op": "cnot", "ctrl": 0, "tgt": 1},
  {"op": "measure", "qubits": [0, 1]}
]
  3. Run it: 
    ./target/release/cqs run --program my_circuit.json --shots 100
That's it! You just simulated a Bell state (quantum entanglement). The output shows you the measurement results.

🎮 Available Operations

cqs-rs supports all the quantum gates you need:

Basic Single-Qubit Gates

Gate What It Does Example
h Hadamard - Creates superposition {"op": "h", "q": 0}
x NOT gate - Flips qubit {"op": "x", "q": 0}
y Pauli-Y gate {"op": "y", "q": 0}
z Phase flip gate {"op": "z", "q": 0}

Rotation Gates (for precise control)

Gate Parameters Example
rx q, theta (angle in radians) {"op": "rx", "q": 0, "theta": 1.57}
ry q, theta {"op": "ry", "q": 0, "theta": 0.785}
rz q, theta {"op": "rz", "q": 0, "theta": 3.14}

Two-Qubit Gates (for entanglement)

Gate What It Does Example
cnot Controlled-NOT - Creates entanglement {"op": "cnot", "ctrl": 0, "tgt": 1}
cz Controlled-Z gate {"op": "cz", "ctrl": 0, "tgt": 1}

Measurement & Special Operations

Operation What It Does Example
measure Measures qubits (gets 0 or 1) {"op": "measure", "qubits": [0, 1]}
postselect Force specific measurement outcome {"op": "postselect", "qubits": [0], "bits": "1"}

📚 Examples

Example 1: Bell State (Entanglement)

Create two entangled qubits:

[
  {"op": "init", "n": 2},
  {"op": "h", "q": 0},
  {"op": "cnot", "ctrl": 0, "tgt": 1},
  {"op": "measure", "qubits": [0, 1]}
]

Result: You'll always get 00 or 11, never 01 or 10!

Example 2: Superposition with Rotation

[
  {"op": "init", "n": 1},
  {"op": "h", "q": 0},
  {"op": "rz", "q": 0, "theta": 1.57},
  {"op": "measure", "qubits": [0]}
]

Example 3: Using a Seed for Reproducibility

[
  {
    "op": "init",
    "n": 3,
    "seed_hex": "0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef"
  },
  {"op": "h", "q": 0},
  {"op": "h", "q": 1},
  {"op": "measure", "qubits": [0, 1, 2]}
]

Run this multiple times - you'll get the same results every time!

Want to see what's happening under the hood?

./target/release/cqs run --program my_circuit.json --shots 10 --log output

This creates detailed logs at output-*.jsonl

🔧 Advanced Features

Constraints

Run simulations with specific quantum state restrictions:

{
  "op": "init",
  "n": 4,
  "constraint": {"kind": "parity", "even": true}
}

Available constraints:

  • {"kind": "none"} - No restrictions (default)
  • {"kind": "hamming", "k": 2} - Exactly k qubits must be |1⟩
  • {"kind": "parity", "even": true} - Even/odd number of |1⟩ states

Noise Models

Simulate real quantum computers with noise:

{"op": "noise", "kind": {"kind": "dephasing", "p": 0.01}, "q": 0}

Types of noise:

  • Dephasing: Random Z gates
  • Depolarizing: Random X, Y, or Z gates
  • Amplitude Damping: Models energy loss

Efficiency Settings

Control memory usage with sparse pruning:

{"op": "init", "n": 10, "eps2": 1e-12}

Smaller eps2 = more accurate but uses more memory

Validation Against Qiskit

Double-check your results against IBM's Qiskit:

python3 scripts/crossval_qiskit.py my_circuit.json

Run Benchmarks

cargo run --release --bin cqs_bench

❓ Need Help?

Here are some tips if things aren't working:

  • JSON errors? Make sure all your brackets and commas are correct
  • Qubit out of range? Remember, if you initialize with n=3, qubits are numbered 0, 1, 2
